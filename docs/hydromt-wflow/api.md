# HydroMT-Wflow API reference

## 1. API reference

The HydroMT-Wflow API provides classes and methods to build, update, and analyze Wflow models. The API is divided into the following sections:

- [SBM model](https://deltares.github.io/hydromt_wflow/stable/api/sbm-model.html#api-wflowsbmmodel): Class for building and managing SBM Wflow models.
- [Sediment model](https://deltares.github.io/hydromt_wflow/stable/api/sediment-model.html#api-wflowsedimentmodel): Class for building and managing sediment transport models.
- [Advanced API](https://deltares.github.io/hydromt_wflow/stable/api/advanced/index.html#api-reference-advanced): In-depth information, scientific motivations and advanced usage.

## 2. WflowSbmModel

The `WflowSbmModel` class represents the main hydrological model implementation using the SBM concept.

Note

Note that this class inherits from [`WflowBaseModel`](https://deltares.github.io/hydromt_wflow/stable/api/advanced/_generated/hydromt_wflow.WflowBaseModel.html#hydromt_wflow.WflowBaseModel "hydromt_wflow.WflowBaseModel") and thereby includes all base functionalities, I/O routines, and setup methods from the base model. Also, note that some setup methods are overridden or extended to include SBM-specific parameters and data. To know more about this, see [WflowBaseModel](https://deltares.github.io/hydromt_wflow/stable/api/advanced/base-model.html#api-base-model).

### 2.4. Component-level API

The table below summarizes the important methods and attributes for each component. These allow for fine-grained reading, writing, modification, and inspection of component data. They are particularly useful when working interactively in Python, for example when updating specific configuration parameters, clipping static maps, or inspecting the forcing data.

Each component exposes a `data` attribute, which holds the underlying model data (e.g. [`dict`](https://docs.python.org/3/library/stdtypes.html#dict "(in Python v3.14)"), [`xarray.Dataset`](https://docs.xarray.dev/en/stable/generated/xarray.Dataset.html#xarray.Dataset "(in xarray v2025.12.0)"), or [`geopandas.GeoDataFrame`](https://geopandas.org/en/stable/docs/reference/api/geopandas.GeoDataFrame.html#geopandas.GeoDataFrame "(in GeoPandas v1.1.2)")), and supports a common set of I/O and manipulation methods such as `read()`, `write()`, and `set()`.

For general I/O at the model level, refer to:`WflowSbmModel` and its `read()` and `write()` methods.

## 3. WflowSedimentModel

The `WflowSedimentModel` class extends Wflow functionality with sediment transport modeling.

Note

Note that this class inherits from [`WflowBaseModel`](https://deltares.github.io/hydromt_wflow/stable/api/advanced/_generated/hydromt_wflow.WflowBaseModel.html#hydromt_wflow.WflowBaseModel "hydromt_wflow.WflowBaseModel") and thereby includes all base functionalities, I/O routines, and setup methods from the base model. Also, note that some setup methods are overridden or extended to include sediment-specific parameters and data. For a full list of inherited methods and attributes, see [WflowBaseModel](https://deltares.github.io/hydromt_wflow/stable/api/advanced/base-model.html#api-base-model).

## 4. Advanced API reference

The advanced HydroMT-Wflow API exposes the underlying classes, workflows, and utility functions used to build, extend, and customize Wflow models. These interfaces are primarily intended for developers and advanced users who want to script model setup workflows, create custom preprocessing routines, or integrate HydroMT-Wflow components into other systems.

The API is organized into the following sections:

- [Base model](base-model.html#api-base-model): Core classes and methods for defining and managing Wflow models.
- [Model components](model-components.html#api-components): Classes and functions for handling specific model components like config, staticmaps, and forcing.
- [Workflows](workflows/index.html#api-workflows): Predefined workflows for common tasks such as setting up basemaps, reservoirs, and calibration.
- [Utilities](utilities.html#api-utils): Helper functions for data processing, file I/O, and other common operations.
